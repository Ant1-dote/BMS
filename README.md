# BMS 项目说明

## 项目简介

本项目是一个基于 Qt 6 的 ADS1256 采样上位机与 STM32 固件联调工程，用于电压采样、继电器控制与充放电循环测试。

核心能力：

- 通过串口控制 STM32 采集 ADS1256 数据
- 在桌面端实时显示单通道/多通道曲线
- 通过 Modbus RTU 控制 2 路继电器（1 路充电、2 路放电）
- 一键启动自动循环充放电
- 数据写入 CSV 与 SQLite，并支持回放分析
- 监控串口、CSV、SQL 缓冲区占用趋势，辅助判断长时间采样瓶颈

当前桌面端工程名为 BMS，CMake 可执行目标为 appBMS。

## 主要功能

- 串口连接与在线状态监控

## UI 使用说明

2.2.2 起，界面采用工业软件常见的“主曲线 + 左侧功能边栏”结构：

- 右侧主区域优先显示实时数据曲线
- 左侧边栏按功能分类：连接采样、运行参数、滤波算法、高级命令、通道管理、缓冲监控
- 右侧底部保留实时数值指标，不遮挡主曲线

### 串口与采样

在 连接采样 区域：

- 连接/断开设备
- 开始连续采样/停止连续采样
- 循环充放电（与采样平级）

点击 循环充放电 时：

- 若当前未采样，会自动启动采样
- 若当前是多通道模式，会自动切换到 SINGLE
- 自动循环将沿用当前运行配置参数

### 运行配置

运行参数区域包含：

- 采样参数（Vref/PGA/DRATE/采样模式等）
- 循环参数（放电终点、充电终点、判定点数、循环次数）

循环参数与运行配置同层管理，修改后立即作用于循环判定。

### 缓冲监控

缓冲监控区域用于判断长时间采样时是否存在处理速度不匹配：

- 串口：串口接收缓冲区占用率
- CSV：CSV 写入缓冲区当前大小
- 本地 SQL：主线程待提交采样缓冲条数
- 异步 SQL：数据库写入线程队列条数

如果某条曲线持续上升并接近上限，说明对应环节处理速度低于数据输入速度；如果曲线周期性升降，说明缓冲区能够正常消化。

### 循环充放电逻辑

- 启动顺序：先放电，再充电
- 放电终点判定：电压 <= 放电阈值，且连续满足 N 个样本
- 充电终点判定：电压 >= 充电阈值，且连续满足 N 个样本
- 最短阶段保护：每阶段至少 3 秒，避免阈值抖动频繁切换
- 循环计数：完成 充电阶段 后计为 1 个完整循环
- 循环次数：0 表示无限循环
- 运行保护：停止采样、断开串口、切换 SCAN8 均会自动停止循环并关闭继电器

### 循环曲线显示

- 每次循环在同一图中叠加一条曲线
- 每条循环曲线的时间从 0 开始
- 不同循环使用不同颜色
- 当前默认显示最近 8 条循环曲线

## STM32 固件侧说明

firmware/stm32.c 提供 ADS1256 采样与串口命令解析，并支持通过 USART2 对继电器板发送 Modbus RTU 写单线圈命令。

### 常用命令示例

```text
START
STOP
RESET
SELFCAL
CFG PSEL=AIN0 NSEL=AINCOM PGA=1 DRATE=0x82 MODE=RDATA ACQ=SINGLE CHMASK=0xFF
CFG ACQ=SCAN8 MODE=RDATA CHMASK=0x0F
RELAY 1 ON
RELAY 1 OFF
RELAY 2 ON
RELAY 2 OFF
RELAY ALL OFF
RELAY ADDR 1
```

### 输出格式示例

```text
AD:123456 HEX:0x01E240
AD8:100,200,300,400,500,600,700,800
RELAY TX: 01 05 00 00 FF 00 8C 3A
RELAY RX: 01 05 00 00 FF 00 8C 3A
```

## 采样与写入稳定性

桌面端采用多级缓冲降低长时间采样卡顿风险：

- 串口接收缓冲：按完整行解析，并限制单次处理时间
- CSV 写入缓冲：批量写入磁盘，停止采样时强制刷新
- SQL 本地缓冲：按批次提交到异步写入线程
- SQL 异步队列：数据库写入线程独立落盘
- 绘图缓冲：实时曲线数据抽稀显示，避免曲线点数无限增长

这些缓冲区只影响处理吞吐和界面响应，不改变原始采样值、CSV 内容和数据库字段含义。

## 常见问题

### 1) 循环启动后继电器不动作

- 先检查 TX/RX 是否交叉连接正确
- 确认继电器地址与 RELAY ADDR 一致
- 查看日志是否有 RELAY TX / RELAY RX / FAIL 信息

### 2) 图上没有采样曲线

- 确认设备已连接且采样已启动
- SINGLE 模式看单通道，SCAN8 模式看多通道
- 检查 DRATE、通道与缩放窗口设置

### 3) 构建后无法启动

- 确认 Qt 运行时完整部署
- 使用 scripts/package.ps1 自动执行 windeployqt

### 4) 长时间采样后界面卡顿

- 打开左侧 缓冲监控，观察串口、CSV、SQL 曲线
- 若异步 SQL 持续增长，说明数据库落盘速度不足
- 若串口缓冲持续增长，说明解析或 UI 主线程处理压力过高
- 若 CSV 缓冲持续增长，优先检查磁盘写入速度和数据目录权限

## 注意事项

- ADS1256 不支持输入引脚对地绝对负电压，负值测量来自差分输入（AINP-AINN）
- 多通道轮询需要关注建立时间与有效吞吐
- 强干扰环境建议优先检查供电、地参考与复位完整性
